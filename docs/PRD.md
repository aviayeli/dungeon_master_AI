# PRD: Dungeon Master AI Game


## תיאור המערכת
משחק הרפתקאות טקסטואלי מבוסס LLM שבו ה-AI משחק בתור מנחה (DM). השחקן כותב פעולות, וה-AI מגיב, מנהל את העולם ומקדם את העלילה בצורה עקבית.

## אילוצים קשיחים (Hard Constraints)
- **מגבלות קוד:** מקסימום 150 שורות לקובץ, מקסימום 20 קבצים בפרויקט.
- **אסינכרוניות:** אסור שה-GUI יקפא בזמן קריאה ל-LLM (חובה Threading/Async).
- **הפרדה מבנית:** הלוגיקה העסקית וה-API חייבים להיות מופרדים לחלוטין מהממשק הגרפי (GUI) באמצעות שכבת SDK.
- **אין Hardcoding:** כל תגובות המשחק מיוצרות דינמית מה-LLM בלבד.

## דרישות ממשק גרפי (GUI) - עברית מלאה
- **סרגל שמאלי:** שם שחקן, כיתה, סרגלי בריאות, מיקום וחפצים.
- **אזור סיפור מרכזי:** בועות שיחה מעוצבות, גלילה אוטומטית למטה, ואנימציית טעינה.
- **שורת קלט:** שדה הזנת טקסט בעברית, כפתור שליחה, וכפתורי פקודות מהירות.
- **פאנל ימני:** סטטיסטיקות, מספר סבבים, לוג אירועים, ועלות API נוכחית.

## לוגיקה ופיצ'רים נדרשים
- **ניהול מצב (State):** בריאות, מיקום, היסטוריה.
- **פקודות מערכת:** תמיכה בפקודות בעברית בלבד: "עזרה", "שמור", "טען", "מלאי", "יציאה".
- **שמירה וטעינה:** היכולת לשמור ולטעון מצב משחק לקובץ JSON.
- **FinOps:** הגבלת חלון ההיסטוריה הנשלח ל-LLM למניעת חריגת עלויות.
- **חיפוש אינטרנט (Web Search Tool):** ה-AI יוכל לחפש מידע אמיתי ולשלב אותו במשחק.
